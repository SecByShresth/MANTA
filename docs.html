üìñ Usage Guide

Step 1: Upload File

Drag and drop a file or click to browse. Supported formats:

  • Windows executables: .exe, .dll, .msi
  • Scripts: .js, .vbs, .ps1, .bat
  • Archives: .zip
  • Android: .apk

Step 2: Review Preview

After upload, you'll see:

  • File name, size, and type
  • SHA256 hash (calculated instantly)
  • Analyze button to start analysis

Step 3: Analyze

Click "Analyze File" to start the analysis process:

  1. Client-Side Analysis: Hashes, strings, entropy, PE parsing
  2. Upload to Backend: (Optional) Send to GitHub Actions
  3. Deep Analysis: YARA scanning, advanced detection
  4. AI Assessment: Threat level evaluation

Step 4: Review Results

Navigate through tabs to view different aspects:

  • Overview: File info and hashes
  • AI Summary: Threat level and recommendations
  • Static Analysis: PE structure (for executables)
  • Strings: Extracted strings with suspicious highlighting
  • Entropy: Visual chart and packing detection
  • YARA Matches: Pattern detection results

Step 5: Export

Download reports in your preferred format:

  • JSON: Machine-readable, complete data
  • HTML: Standalone report with styling
  • Text: Simple text format

üé® Understanding the Analysis

Hash Values

Cryptographic hashes uniquely identify files. Use them to:

  • Search on VirusTotal or other threat intelligence platforms
  • Verify file integrity
  • Track malware samples

Entropy Analysis

Entropy measures randomness in data:

  • 0-4: Highly structured (normal code/text)
  • 4-6: Normal executable data
  • 6-7: Compressed or mixed data
  • 7-8: Likely encrypted or packed

High entropy (>7.0) often indicates packing or encryption, common in malware.

String Analysis

Extracted strings can reveal:

  • URLs and IP addresses (C2 servers)
  • File paths (persistence locations)
  • API calls (malicious behavior)
  • Registry keys (system modifications)

PE Structure

For Windows executables, PE analysis shows:

  • Sections: Code, data, resources
  • Imports: External functions used
  • Characteristics: File properties
  • Timestamp: Compilation date

AI Assessment

The AI model evaluates:

  • Threat Level: Low, Medium, High
  • Behaviors: Suspicious activities detected
  • Confidence: How certain the assessment is
  • Recommendations: Next steps for analysis

üêõ Troubleshooting

Common Issues

Q: Hashes not calculating

A: Ensure you're using a modern browser with Web Crypto API support (Chrome, Firefox, Edge, Safari).

Q: PE parsing fails

A: PE parsing only works for valid Windows executables (.exe, .dll). Other file types will skip this step.

Q: Backend analysis not working

A: Check that:

  • GitHub token is valid and has correct permissions
  • Workflow file exists in .github/workflows/
  • Repository has GitHub Actions enabled
  • Check the Actions tab in your repository for errors

Q: Page not loading on GitHub Pages

A: Wait 2-3 minutes after enabling Pages. Verify repository is public or token has access.
